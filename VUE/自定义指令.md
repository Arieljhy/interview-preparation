要灵活创建自定义指令，你需要理解它的生命周期钩子和钩子函数的参数。

生命周期钩子函数
指令的生命周期钩子与组件类似，但作用是绑定指令的DOM元素。最常用的是：
- mounted：当绑定元素被插入父节点时调用（最常用）。
- updated：当组件所在的VNode更新时调用。
- unmounted：当指令与元素解绑且父组件已卸载时调用，用于清理工作（如取消事件监听）。

钩子函数的参数
每个钩子函数都会接收以下参数：
el：指令绑定到的DOM元素，可以直接操作它。
binding：一个对象，包含以下有用的属性：
- value：传递给指令的值。例如 v-my-dir="123"，value 就是 123。
- arg：传递给指令的参数。例如 v-my-dir:foo，arg 就是 "foo"。
- modifiers：一个包含修饰符的对象。例如 v-my-dir.bar.baz，modifiers 是 { bar: true, baz: true }。

1. Hash 模式

URL 中 hash 的变化不会导致页面刷新。

通过 window.location.hash 获取或设置 hash。
监听 hashchange 事件来响应 hash 的变化。
服务器配置：由于 hash 部分不会被发送到服务器，所以服务器不需要做任何特殊处理。

1. History 模式
原理
History 模式利用 HTML5 History API（pushState、replaceState 和 popstate 事件）来实现路由。这种模式允许我们改变 URL 而不重新加载页面，同时还可以更新浏览器的历史记录。

实现要点
使用 history.pushState() 和 history.replaceState() 来添加和修改历史记录条目。
监听 popstate 事件来响应浏览器的前进和后退操作。
服务器配置：因为用户可能直接访问任何一个 history 模式下的 URL，所以服务器需要配置为对于所有路径都返回同一个 HTML 页面（通常是首页），然后由前端路由来处理具体的路径。

#### 了解react底层吗（fiber）

#### fiber是如何做到中断执行的（时间分片）
Fiber架构通过将渲染任务拆分成多个小任务（Fiber节点），利用浏览器的空闲时间执行这些任务，并通过优先级调度实现中断和恢复，从而实现了时间分片。这使得React能够在大规模更新时保持界面的响应性，不会阻塞用户交互。

#### `从setState到页面rerender中间发生了什么`
setState时候会把更新放入更新队列，触发单个更新 或者 批量更新；在协调阶段，根据更新生成新的Fiber tree，和旧的Fiber tree对比，找到对应的diff；在提交阶段，把差异触发真实Dom的更新，最终渲染到浏览器中。

#### 讲一讲react的事件代理
利用事件委托（事件代理）的思想, React 会把所有事件监听器统一绑定到最外层容器（React 17 之前委托到document上、17之后委托到root节点）上，通过事件冒泡捕获所有子组件事件，然后在内部合成事件系统中进行处理。
`SyntheticEvent（合成事件） ，是 React 封装的原生事件对象。合成事件提供统一接口，并模拟了原生事件的冒泡和捕获行为。你仍可以通过 event.nativeEvent 获取原生事件对象。`

React 的事件代理机制优势：
✅ 跨浏览器兼容：使用合成事件（SyntheticEvent）系统做跨浏览器兼容处理。
✅ 性能更好：减少了事件监听器的数量，特别是组件较多时。
✅ 便于事件回收：当组件卸载时不需要显式解绑 DOM 事件，React 自动管理。
